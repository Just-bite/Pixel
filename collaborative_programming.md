# Опыт парного программирования

В ходе разработки ПО команда проекта неоднократно сталкивается с объемными задачами, решение которых требует проанализировать  мнение как минимум нескольких разработчиков.
В качестве стратегий парного программирования для проекта были выбраны стратегии 'На равных", а также удаленное парное программирование.

В качестве примера реализуемой задачи может выступить разработка правой панели GUI, включавшей в себя панель выбора цвета и слоев. Разработчики последовательно реализовали каркас, а затем удаленно параллельно реализовали функционал слоев (разработчик 1) и функционал палитры (разработчик 2)
После живой встречи и обсуждения кода ветки были объединены.
Такой подход является наиболее подходящим для небольшого проекта с возможностью четкого разделения ответственности и непрерывным циклом интеграции.
Проблем по ходу работы не возникло.

Команда проекта активно использует ПП для иных задач по другим дисциплинам, таких как разработка сетевого программного обеспечения, цифровая обработка сигналов
